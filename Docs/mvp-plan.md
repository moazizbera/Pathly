# MVP Plan

## Goal

Deliver a polished hackathon MVP of Pathly that demonstrates role-based productivity planning for three user categories: Student, Employee, and Teacher.

## Build Order

### Phase 1: Foundation

- Initialize Next.js app with TypeScript and Tailwind CSS
- Configure Supabase project
- Add authentication
- Create base layout and navigation

### Phase 2: Core User Data

- Build onboarding flow
- Save category-based profile data
- Add profile editing

### Phase 3: Planning System

- Add goals and tasks CRUD
- Add due dates and estimated effort
- Create shared dashboard layout

### Phase 4: Personalization

- Add category-aware dashboard cards
- Build rule-based recommendation engine
- Show one recommended next action with explanation

### Phase 5: Polish

- Improve visuals and onboarding copy
- Add empty states and success states
- Prepare live demo data
- Record demo video and finalize submission assets

## MVP Scope Checklist

- User sign-up and login
- Category selection
- Role-aware profile
- Task and goal management
- Personalized dashboard
- Next-step recommendation

## Stretch Goals

- Google sign-in
- AI-generated task breakdown
- Habit tracking
- Weekly summary

## Demo Script Outline

1. Introduce the problem with generic productivity tools.
2. Sign up as a user.
3. Select a category.
4. Add tasks and deadlines.
5. Show the personalized dashboard.
6. Show the recommended next action and explanation.
7. Close with the broader vision for more categories.

## Delivery Standard

The MVP should feel focused, reliable, and visually intentional. One polished flow is more valuable than many incomplete features.
