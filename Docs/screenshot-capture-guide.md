# Pathly Screenshot Capture Guide

Use this guide to capture the final screenshot set for the demo video and NotebookLM prompt.

## Capture Rules

- use desktop width, not mobile
- keep browser zoom at 100%
- hide bookmarks and personal tabs
- keep the same browser size across all captures
- prefer clean crops over tiny full-page screenshots
- export PNG files
- use the exact filenames below

## Screenshot Order

### 1. `01-home-hero.png`

- Route: `/`
- Sign-in state: before sign-in
- Capture: the top hero section of the landing page
- Must show:
  - Pathly brand and header
  - main headline
  - supporting line about clear planning and reduced context switching
  - main CTA area
- Why it matters:
  - this is the first frame of the story and should immediately position Pathly as more than a generic planner

## Capture Session Checklist

- prepare realistic sample data before dashboard screenshots
- make sure at least two roles are enabled for the same account
- keep one screenshot in focused role mode and one in all-roles mode if possible
- open the AI Suggest dialog before capturing screenshot 8
- verify every filename exactly matches this guide
