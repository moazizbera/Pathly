# Pathly NotebookLM Video Prompt

Live app: https://pathly.maziz-abdelrahman.workers.dev/

Use this file as the source prompt for NotebookLM after you capture the final screenshots listed in `screenshot-capture-guide.md`.

## Video Goal

The video should make one point very clearly:

Pathly is not another task manager. It is a multi-role planning system that helps one person prioritize across different responsibilities, understand why the next action matters, and switch between focused role mode and an all-roles view.

## Recommended Format

- length: 90 to 120 seconds
- pacing: 10 screenshots
- voice: clear, confident, practical
- style: no hype, no filler, no generic AI language

## Required Input Assets

Before using NotebookLM, prepare:

- the 10 screenshots listed in `screenshot-capture-guide.md`
- the exact filenames from that guide
- the short caption for each screenshot
- the final closing line from this document

## What NotebookLM Should Understand

NotebookLM should describe the product in this order:

1. Pathly solves the problem of flat planning tools.
2. One person can have multiple roles.
3. Pathly stores one base profile and supports multiple role profiles.
4. Tasks are role-owned first, then aggregated into an all-roles view.
5. The dashboard recommends the next move and explains why.
6. The role switcher changes the planning context visibly.
7. AI suggestions, task list, and calendar all use the same role language.

## Copy-Paste Prompt For NotebookLM

Paste the block below into NotebookLM together with the screenshots.

```text
Create a polished 90-120 second product video voiceover for Pathly.

Pathly is a multi-role productivity platform with the tagline: Smart planning for every role.

Core message:
Pathly helps overloaded people move from a messy list of responsibilities to one clear next step, across every role they live in.

Important product truth:
- This is not a generic to-do app.
- One user can hold multiple roles such as Student, Employee, and Teacher.
- Pathly keeps one shared profile, multiple role profiles, and tasks that are role-owned, shared, or general.
- The dashboard can switch between a focused role view and an all-roles view.
- The system explains why the recommended next move matters.

Use the screenshots in the exact order provided.
For each screenshot:
- describe what the viewer is seeing
- explain why that screen matters in the product story
- keep the narration tight and presentation-ready

Tone requirements:
- confident
- practical
- product-demo ready
- no exaggerated claims
- no generic AI buzzwords

Narration structure:
1. open with the problem of flat productivity tools
2. explain why multi-role planning matters
3. show how Pathly introduces role-aware onboarding
4. explain the dashboard recommendation and visible AI reasoning
5. explain focused role mode versus all-roles mode
6. show that role plans, suggestions, task list, and calendar all stay consistent
7. close with Pathly as a calmer planning experience

Do not invent features that are not shown in the screenshots.
Do not mention implementation technologies.
Do not sound like an ad. Sound like a sharp product demo voiceover.

End with this exact final line:
Pathly helps overloaded people move from a messy list of responsibilities to one clear next step, across every role they live in.
```
