# Pathly NotebookLM Video Prompt

Live app: https://pathly.maziz-abdelrahman.workers.dev/

Use this file as the source prompt for NotebookLM after you capture the final screenshots listed in `screenshot-capture-guide.md`.

## Video Goal

The video should establish one clear idea:

Pathly is not a generic planner. It is a multi-role execution system that helps one person manage overlapping responsibilities, understand the next best action, and move between focused role planning and an aggregated all-roles view.

## Recommended Format

- length: 90 to 120 seconds
- pacing: one narrative pass across 10 screenshots
- voice: sharp, confident, product-demo ready
- style: practical, credible, and judge-friendly

## Required Input Assets

Before using NotebookLM, prepare:

- the 10 screenshots listed in `screenshot-capture-guide.md`
- the exact filenames from that guide
- the final closing line from this document

## What NotebookLM Must Understand

NotebookLM should make these ideas obvious:

1. Flat task managers break down when one person operates in multiple roles.
2. Pathly is built around one shared user profile plus multiple role profiles.
3. Tasks are role-owned first, then surfaced in focused or aggregated views.
4. The dashboard recommends a next move and explains the reasoning behind it.
5. The role switcher changes the planning context in a visible, useful way.
6. AI suggestions, tasks, calendar, and profile settings all use the same role model.
